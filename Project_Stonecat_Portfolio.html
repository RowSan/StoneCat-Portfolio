CASE STUDY — AI SYSTEMS ARCHITECTURE — 2024

PROJECT STONECAT

A ground-up AI-powered inventory intelligence system — architected, engineered, and deployed as a local-first, zero-dependency platform eliminating cloud AI costs, legacy integration failures, and manual data workflows.

Role Lead AI Solutions Architect
Client SMG Stone Gallery, Sydney
Timeline 2024 – Present
Type Full-Stack AI System

The Problem

⬡ Before — System Failures
  • Legacy ERP (Fishbowl) exposed an unreliable REST/XML socket API — frequent sync failures caused stale inventory data across the business
  • Recurring cloud AI API costs for data processing with no guarantee of data privacy or sovereignty
  • Unstructured product data (free-text "Stone Codes") made filtering, searching, and catalogue management impossible
  • Pagination artifacts, metadata inconsistencies, and pricing logic conflicts corrupted the data layer
  • Sales teams had no real-time inventory access in the field — reliant on manual lookups and static reports
  • No automation: every data sync was a manual process requiring human intervention
⬡ After — System Capabilities
  • Fishbowl API bypassed entirely — replaced with a resilient CSV-to-Supabase bridge with zero sync failures
  • Local-first LLM (DeepSeek-R1 via Ollama) deployed on-premises — zero cloud AI cost, full data sovereignty
  • Proprietary Stone Code parser decomposes unstructured identifiers into structured relational attributes enabling advanced filtering
  • Automated multi-stage cleaning pipeline eliminates all pagination artifacts, metadata drift, and pricing conflicts
  • React Native mobile app gives sales team real-time inventory access from any device, anywhere
  • macOS Launchd automation delivers 24/7 background sync with self-recovery on failure

SYSTEMS ARCHITECTURE

Fishbowl API Bypass — Legacy Integration Strategy

The ERP vendor's REST/XML socket API was fundamentally unreliable — sync failures were routine and unresolvable through conventional debugging. Rather than continuing to fight an unmaintainable integration, the decision was made to architect a bypass entirely. This involved reverse-engineering the ERP's CSV export capability, designing a deterministic file-watch pipeline, and building a direct CSV-to-Supabase ingestion bridge. The result: a simpler, faster, and completely reliable data pathway that outperforms the official API in every measurable dimension.

Key insight: sometimes the correct architectural decision is to eliminate the dependency entirely, not to fix it.

04
PARSING ENGINE

Stone Code Decomposer

Built a proprietary parsing engine to decompose free-text product identifiers ("Stone Codes") into four structured relational attributes: Name, Material, Finish, and Thickness. This converted an unsearchable catalogue into a fully queryable, filterable data structure — unlocking advanced inventory analytics that were previously impossible.

Custom parser String decomposition Structured attributes Catalogue intelligence

MOBILE / SECURITY

Mobile Integration & Security Layer

Integrated the entire backend system with a React Native TypeScript mobile application delivering real-time inventory access for sales teams. Applied production security hardening: Supabase JWT key rotation, environment variable isolation, and vulnerability remediation — resulting in a secure, audit-ready deployment.

Architectural Decisions

Decision Alternative Rejected Choice Made Rationale
AI Processing OpenAI / Anthropic cloud API Local LLM (DeepSeek-R1 via Ollama) Eliminates recurring cost, ensures data stays on-premise, no API rate limits or outage dependency
ERP Integration Fix & maintain Fishbowl REST/XML API Full API bypass via CSV bridge API was architecturally unreliable; bypass was simpler, faster, and more maintainable
Database On-premise MySQL / SQLite Supabase (hosted PostgreSQL) Real-time capabilities, REST API out of the box, React Native SDK, JWT auth — at near-zero cost
Scheduling Cron jobs / manual triggers macOS Launchd daemons Native OS-level process management with automatic restart on failure — more reliable than cron in practice
Mobile Stack Native iOS / Android or PWA React Native (TypeScript) Single codebase, strong TypeScript safety, Supabase client compatibility, fast iteration
Data Parsing Manual taxonomy / spreadsheet lookup Custom Stone Code parser engine Automated, scalable decomposition of all future inventory data without human review
